# React Frontend Developer

Skill untuk development React JS frontend aplikasi.

## Triggers
- react
- frontend
- component
- hook
- jsx
- typescript react
- redux
- state management

## Instructions

Ketika user meminta bantuan terkait React development, gunakan keahlian berikut:

### Core React
- Functional components dengan Hooks (useState, useEffect, useContext, useReducer, custom hooks)
- Class components dengan lifecycle methods
- Component composition dan reusability
- Props dan state management

### State Management
- Context API dan useReducer
- Redux / Redux Toolkit (actions, reducers, store, middleware)
- Zustand, Recoil, atau Jotai untuk atomic state
- MobX untuk observable state

### Styling
- CSS-in-JS (Styled-components, Emotion)
- CSS Modules
- Utility-first (Tailwind CSS)
- Component libraries (MUI, Ant Design, Chakra UI)
- Framer Motion untuk animasi

### Build & Tools
- Webpack configuration
- Vite untuk fast development
- Babel transpiling
- ESLint & Prettier
- npm/yarn/pnpm package management

### Testing
- Jest & React Testing Library untuk unit testing
- Cypress/Playwright untuk E2E testing
- Storybook untuk component documentation

### Performance
- Code splitting & lazy loading
- Memoization (React.memo, useMemo, useCallback)
- Virtualization untuk long lists
- Core Web Vitals optimization

### API Integration
- RESTful APIs dengan Fetch/Axios
- GraphQL dengan Apollo Client
- Authentication (JWT, OAuth)
- Error handling & interceptors

### Security Best Practices
- Input validation & sanitization
- XSS prevention (hindari dangerouslySetInnerHTML)
- CSRF protection
- Secure token storage
- Environment variables (tidak expose secrets)
- Dependency security audits

### Validation
- Client-side validation (Yup, Zod, Joi)
- Form handling (Formik, React Hook Form)
- PropTypes atau TypeScript

### Error Handling
- Error Boundaries untuk catching component errors
- Error tracking (Sentry, LogRocket)
- API error handling dengan user-friendly messages

## Guidelines
1. Selalu ikuti clean code principles
2. Gunakan TypeScript jika project mendukung
3. Implement accessibility (ARIA, WCAG)
4. Write maintainable & documented code
5. Follow project conventions yang ada
